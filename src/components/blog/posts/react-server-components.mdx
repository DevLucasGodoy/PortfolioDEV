Os **React Server Components (RSC)** representam uma das maiores mudanças na arquitetura React desde a introdução dos Hooks. Com eles, é possível renderizar componentes diretamente no servidor — sem enviar JavaScript desnecessário para o cliente.

## O que são Server Components?

Server Components são componentes que executam **exclusivamente no servidor**. Eles têm acesso direto a bancos de dados, sistemas de arquivos e APIs internas, sem expor essas operações ao navegador.

```tsx
// Isso roda só no servidor
async function UserProfile({ id }: { id: string }) {
  const user = await db.users.findById(id); // acesso direto ao banco

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
}
```

## Vantagens principais

- **Bundle menor**: o código do servidor não vai para o cliente
- **Acesso direto a dados**: sem precisar de APIs intermediárias
- **SEO nativo**: HTML gerado no servidor desde o início
- **Segredos protegidos**: chaves de API nunca chegam ao browser

## Server vs Client Components

| Característica              | Server Component | Client Component |
| --------------------------- | ---------------- | ---------------- |
| Hooks (useState, useEffect) | ❌               | ✅               |
| Acesso a banco de dados     | ✅               | ❌               |
| Interatividade              | ❌               | ✅               |
| Tamanho no bundle           | 0kb              | Incluso          |

## Quando usar cada um?

Use **Server Components** para busca de dados, layouts estáticos e conteúdo que não precisa de interatividade. Use **Client Components** para formulários, animações e qualquer coisa que dependa do estado do usuário.

A combinação dos dois é o que torna o Next.js App Router tão poderoso — e entender essa fronteira é essencial para construir aplicações performáticas em 2025.
