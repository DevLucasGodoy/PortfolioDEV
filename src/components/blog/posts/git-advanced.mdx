Todo desenvolvedor conhece o trio `add`, `commit` e `push`. Mas o Git tem um arsenal de ferramentas que podem salvar horas de trabalho — ou um projeto inteiro. Veja os que mais fazem diferença.

## git stash — guarde mudanças sem commitar

Precisa trocar de branch mas não quer commitar um trabalho incompleto?

```bash
git stash push -m "feature: formulário de login incompleto"
# troca de branch, faz o que precisava...
git stash pop  # de volta ao trabalho
```

## git bisect — encontre o commit que quebrou tudo

O `bisect` faz uma busca binária no histórico para achar exatamente onde um bug foi introduzido:

```bash
git bisect start
git bisect bad                  # commit atual está quebrado
git bisect good v1.2.0          # essa versão estava boa
# o Git vai apontando commits, você testa e marca good/bad
git bisect good  # ou: git bisect bad
```

## git reflog — o histórico que salva vidas

O `reflog` guarda **tudo** que aconteceu no repositório local, mesmo coisas que não aparecem no `git log`:

```bash
git reflog
# encontre o hash do commit "perdido"
git checkout -b recuperado abc1234
```

Deletou uma branch sem querer? O reflog tem o commit.

## git rebase interativo

Antes de abrir um PR, organize seus commits com `rebase -i`:

```bash
git rebase -i HEAD~4
```

Você pode reordenar, juntar (`squash`), editar mensagens ou até remover commits — deixando o histórico limpo e compreensível para o time.

## git worktree — duas branches ao mesmo tempo

Trabalhe em duas branches simultaneamente sem precisar clonar o repositório de novo:

```bash
git worktree add ../projeto-hotfix hotfix/bug-critico
# agora você tem duas pastas do mesmo repo, cada uma em uma branch
```

Dominar o Git vai muito além dos comandos básicos — e investir algumas horas aprendendo esses recursos retorna em produtividade e muito menos estresse em situações de crise.
